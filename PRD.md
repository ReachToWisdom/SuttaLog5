# PRD: 빠알리경전학습 앱

## 목표
빠알리 경전 본문 자체를 교재로 삼아, 음절 → 단어 → 어근까지 분해해 학습하는 모바일 우선 웹앱.

## 핵심 컨셉
- **경전 = 교재**. 별도 문법 강의 없이, 본문 위에서 직접 빠알리어를 익힌다.
- 본문/단어 모두 **생략 없이 100% 노출**.
- 반복 학습 부담은 "노출 횟수 제한"으로 사용자가 제어.

## 핵심 기능

### F1. 경전 본문 분해 표시
- 빠알리 원문을 **음절 단위**로 끊어 표시.
- 단어 클릭/탭 시 **단어 + 어근 + 격변화 + 뜻** 펼침.
- 9개 경전 전문 수록 (DOCS 폴더 기준).
- **데이터는 사전 처리된 JSON**. 런타임 AI 분석 X.

### F2. 반복 노출 제어 (환경설정)
- 동일 **문법** N회까지만 노출, 초과분 생략 (1 ≤ N, 사용자 지정).
- 동일 **단어/숙어**도 같은 방식.
- "생략" = 분해/주석을 안 보임. 본문 자체는 항상 노출.

### F3. 독해 가림
- 한국어 독해는 **기본 가림**.
- **문장 단위** 또는 **페이지 단위** 펼치기 (환경설정에서 단위 선택).

### F4. 페이지네이션
- **고정 분량 페이지네이션** (콘텐츠 기준).
- 문장 길이 가변 → 분해/독해까지 한 화면에 답답하지 않은 분량으로 자동 분할.
- 페이지에는 **고유 ID** 부여 (예: `M10-p015`). 단말 무관 동일.
- 화면 크기에 따라 **폰트 크기만 가변**, 페이지 경계는 불변.

### F5. 페이지별 메모 + GitHub 저장
- 모든 페이지에 메모 입력 칸.
- 작성 후 저장 → **GitHub 레포에 자동 commit**.
- 메모 데이터 구조 (페이지당 1개 JSON):
  ```
  page_id, 원본 페이지 스냅샷, 사용자 메모 본문,
  상태(작성중/검토대기/승인/보류/재요청),
  수정 제안 diff(before/after), 승인 이력(날짜)
  ```

### F6. 일괄 수정 사이클
- 트리거: **사용자가 Claude Code에 직접 "메모 확인해서 수정해줘"라고 말함** (앱 내 버튼 X).
- Claude Code가 GitHub의 메모를 읽고 → 본문 수정 PR 생성.
- 앱이 PR 감지 → 해당 페이지 메모에 "수정 제안 도착" 표시.
- **원본 / 수정안 좌우 비교** UI.
- 사용자가 **승인 / 보류 / 재요청** 선택.
  - 승인 → main 머지 + `approvals.json`에 `{날짜, page_id, commit_hash}` 누적.
  - 보류 → 대기 상태 유지.
  - 재요청 → 메모에 추가 코멘트 → 다시 의뢰.

## 비기능 요구사항
- **모바일 우선**: 스마트폰 화면 기준 UI.
- **기기/OS 독립**: 표준 브라우저 동작이면 모두 OK.
- **5년 후에도 동작 보장**: 빌드도구/프레임워크 의존성 0.

## 기술 스택
- **Vanilla HTML + CSS + JavaScript** (프레임워크 없음, 빌드 없음).
- **GitHub Pages** 호스팅.
- 데이터: 사전 처리된 JSON (9개 경전 분해 결과).
- 메모: GitHub REST API (사용자의 토큰)로 직접 commit.
